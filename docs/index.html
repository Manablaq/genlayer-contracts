// 01 — known bugs
Things that silently break.

None of these produce a useful error message. Every GenLayer developer hits them. All five were discovered building the four oracle contracts.

01
web.get() stores nothing

Transaction finalizes. No error thrown. State is always an empty string. Use web.render() exclusively.

02
TreeMap doesn't persist

Works in local testing. State vanishes after every transaction on Studio. Use str + JSON.

03
sender_address kills deploy

gl.message.sender_address in __init__ causes a runtime error at deploy time.

04
run_nondet_unsafe + web

Combining run_nondet_unsafe with any web fetch fails at runtime. Use prompt_comparative.

05
gl.vm.UserError in __init__

Raised inside __init__ causes deploy execution failure. Use assert instead.

Bug 01 — web.get() silent failure

The most dangerous bug because it produces zero errors. The transaction finalizes and your state stores an empty string every time.

❌ Incorrect — silent data loss
# Looks correct. Compiles. Deploys. Finalizes. Stores nothing.
response = gl.nondet.web.get(url)
self.store = response.body  # Always empty string on Studio
✅ Correct — from CryptoOracle
def fetch() -> str:
    try:
        raw = gl.nondet.web.render(url, mode="text")
        if not raw or raw.strip() == "null":
            return json.dumps({"error": "API down.", "status": "unavailable"})
        data = json.loads(raw)
        return json.dumps({"price": data["lastPrice"], "status": "ok"}, sort_keys=True)
    except Exception:
        return json.dumps({"error": "API down.", "status": "unavailable"})

fresh = gl.eq_principle.prompt_comparative(fetch, "...")

Bug 02 — TreeMap state vanishes

❌ Incorrect — state lost on Studio
class CryptoOracle(gl.Contract):
    prices: TreeMap[str, str]  # Disappears after each transaction
✅ Correct — used across all 4 contracts
class CryptoOracle(gl.Contract):
    store: str  # Persists correctly on Studio
    pulse: str  # Freshness metadata

def __init__(self):
    self.store = "{}"
    self.pulse = "{}"

Bug 03 — sender_address at deploy time

❌ Incorrect — runtime error on deploy
def __init__(self):
    self.owner = gl.message.sender_address  # Runtime error on deploy
✅ Correct — from MultiKeyVault
def __init__(self, owner_address: str):
    assert is_valid_address(owner_address), "Invalid owner address."
    self.owner           = owner_address
    self.allowed_callers = json.dumps([owner_address])
// 02 — patterns
What actually works.

Calibrate your equivalence string

The second argument to prompt_comparative defines what validators consider "equivalent." Write it to match your data's actual volatility — not copied from another contract.

CryptoOracle — price data, allow 2% variance
"They are equivalent if price values are within 2% and symbol matches."
WeatherOracle — sensor data, per-field tolerances
"They are equivalent if temperature is within 1 degree, humidity within 5%, and wind speed within 3 km/h."
WeatherOracle forecast — wider tolerance
"They are equivalent if temperatures are within 2 degrees and dates match exactly."
SocialOracle trending / latest — count-based match
Trending: "at least 3 of the same story titles." Latest (more volatile): "at least 2 of the same story titles or IDs."
Rule: Too strict → consensus fails on valid data. Too loose → bad data passes. Match the threshold to how much your data can legitimately differ between two validator calls made seconds apart.

Always use sort_keys=True

Validators independently build Python dicts. Without sort_keys=True, key ordering is non-deterministic and consensus fails on identical data.

# Every state write in all 4 contracts
self.store = json.dumps(all_data, sort_keys=True)
self.pulse = json.dumps(hb, sort_keys=True)

Booleans and integers are strings

All state fields must be typed as str. From MultiKeyVault:

class MultiKeyVault(gl.Contract):
    is_paused:   str  # "true" or "false" — NOT bool
    rate_limit:  str  # "100" — NOT int
    key_version: str  # "1", "2", "3" — NOT int

# Comparing
assert self.is_paused == "false", "Vault is already paused."
# Incrementing
self.key_version = str(int(self.key_version) + 1)

Module-level functions work — use them

Define shared fetch logic outside the class and call from inside closures. Used across SocialOracle and WeatherOracle.

# Outside the class
def _pull_stories(feed: str, limit: int = 10) -> str:
    ids_raw = gl.nondet.web.render(
        f"{HN_API}/{feed}stories.json", mode="text"
    )
    # Multiple web.render() calls inside one closure are fine
    for story_id in ids[:limit]:
        raw = gl.nondet.web.render(f"{HN_API}/item/{story_id}.json", mode="text")
    return json.dumps({"items": items}, sort_keys=True)

class SocialOracle(gl.Contract):
    @gl.public.write
    def fetch_trending(self) -> typing.Any:
        def fetch() -> str:
            return _pull_stories("top", 10)
        fresh = gl.eq_principle.prompt_comparative(fetch, "...")

Closures in loops — a hidden bug risk

Defining a closure inside a for loop captures the variable reference, not its value. Use default argument binding to capture values safely.

⚠️ Risky — variable capture
for symbol in symbols:
    url = f"https://api.binance.com/...?symbol={symbol}"
    def fetch() -> str:
        return gl.nondet.web.render(url, ...)  # which url?
✅ Safe — default argument binding
for symbol in symbols:
    url = f"https://api.binance.com/...?symbol={symbol}"
    def fetch(u=url) -> str:  # captures value at definition
        return gl.nondet.web.render(u, ...)

Structure AI prompts for specific JSON output

Always define an exact response schema in the prompt. Write equivalence checks that target the key decision only. From WeatherOracle:

prompt = (
    f"Analyze weather for {city}. Respond ONLY with JSON:\n"
    '{"alert_level": "safe" or "caution" or "danger", '
    ' "reason": "one sentence explanation"}'
)
result = gl.nondet.exec_prompt(prompt)

gl.eq_principle.prompt_comparative(
    analyze,
    "Equivalent if they assign the same alert_level."
)

Always handle errors inside the closure

def fetch() -> str:
    try:
        raw = gl.nondet.web.render(url, mode="text")
        if not raw or raw.strip() in ["null", ""]:
            return json.dumps({"error": "API down.", "status": "unavailable"})
        # parse and return data
    except Exception:
        return json.dumps({"error": "API down.", "status": "unavailable"})

Use gl.vm.UserError in write methods for preconditions

@gl.public.write
def generate_briefing(self) -> typing.Any:
    data = json.loads(self.store)
    if "trending" not in data:
        raise gl.vm.UserError("No stories cached. Call fetch_trending first.")

Only use public APIs

Validators need to independently replicate every web call. Auth-required APIs cannot reach consensus.

✅ Compatible public APIs
Binance public ticker · Open-Meteo · Hacker News Firebase · CoinGecko public · Any fully public REST API
❌ Incompatible APIs
APIs requiring OAuth tokens · API keys in headers · HMAC request signing · Login-protected endpoints

Use sentinel values for freshness tracking

# Dict-like state
self.pulse = "{}"

# To distinguish "never fetched" from "fetched but empty" — used in SocialOracle
self.pulse = "never"

if self.pulse == "never":
    return json.dumps({"fresh": False, "reason": "Cache never populated."})

// 04 — known limitations
Platform limits.

These are confirmed Studio constraints — not bugs in your contract code.

  • CONFIRMED
    web.get() + response.body returns emptySilent data loss. No error thrown. Workaround: use web.render() exclusively.
  • CONFIRMED
    TreeMap state does not persist on StudioState lost after every transaction. Workaround: use str fields with JSON serialization.
  • CONFIRMED
    run_nondet_unsafe fails with web fetchingCannot combine unsafe nondeterminism with web calls. Workaround: use prompt_comparative.
  • CONFIRMED
    Private APIs incompatible with consensusValidators can't replicate auth-required calls independently. Workaround: public APIs only.
  • CONFIRMED
    gl.message.sender_address in __init__ failsCauses runtime error at deploy time. Workaround: pass caller address as constructor parameter.
  • NOTE
    Leader rotation shows more than 5 validatorsNormal behavior — reflects cumulative leaders across consensus rounds, not the validator set size.
